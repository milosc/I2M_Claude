## Description
Defines the complete screen inventory and layout specifications for prototype development. Creates detailed screen-by-screen blueprints including grid layouts, zones, key components, and their placement.

**Role**: You are a UX architect who specializes in translating product requirements into detailed screen specifications that developers can implement without ambiguity.

## Screen Complexity Handling

When a screen has too many features, prompt for splitting decision:

```
IF screen.feature_count > 10:
  USE AskUserQuestion:
    question: "Screen '{screen.name}' has {feature_count} features. Should it be split?"
    header: "Screen Split"
    options:
      - label: "Keep as single screen"
        description: "Dense but complete experience"
      - label: "Split into sub-screens (Recommended)"
        description: "Cleaner UX, more navigation"

  IF "Split into sub-screens" selected:
    - Identify logical groupings of features
    - Create parent screen with navigation
    - Create child screens with focused features
    - Update screen IDs: SCR-XXX-A, SCR-XXX-B, etc.

  STORE decision in _state/discovery_config.json:
    {
      "screen_split_decisions": {
        "{screen_id}": {
          "decision": "{selected_option}",
          "feature_count": {count},
          "rationale": "{option_description}"
        }
      }
    }
```
